# Internationalization (i18n)

This directory contains all raw translation assets and the tooling required to process them for the developer documentation site.

## Directory Structure

*   **`raw/`**: Contains the raw, unzipped exports from the Google Localizer tool.
    *   These folders are named by date and project ID (e.g., `2024-12-21_project_4198229`).
    *   These exports are **incremental**. Newer folders contain new translations or updates to existing ones.
    *   **Do not modify files here directly.** This is the source of truth from the translation vendor.

*   **`consolidated/`**: The output directory generated by `scripts/consolidate_translations.py`.
    *   This directory represents the current state of the localized site.
    *   It mimics the structure of the root documentation folder (e.g., `docs/guides/...`).
    *   It is safe to delete this directory; it can always be rebuilt from `raw/`.

*   **`scripts/`**: Contains Python scripts for managing translations.
    *   `consolidate_translations.py`: The main script used to merge `raw/` into `consolidated/`.

## The Consolidation Process

The `consolidate_translations.py` script performs the following actions:

1.  **Cleans**: Deletes the `consolidated` directory to start fresh.
2.  **Sorts**: Identifies export folders in `raw/` and processes them in chronological order (oldest to newest). This ensures the latest translations overwrite older ones.
3.  **Normalizes**:
    *   It strips internal prefix folders found in the exports (e.g., `outline/` or `0outline/`) to match the root project structure.
    *   It maps the locale code (e.g., `es-419`, `zh-CN`) to the top level of the `consolidated` directory.

### File Handling

*   **`.md.html` Files**:
    *   These are the translated versions of the original Markdown (`.md`) files.
    *   The translation tool exports them as HTML.
    *   The script copies them **as-is**, preserving the `.md.html` extension.
    *   These files contain the actual localized content for pages like guides and reference docs.

*   **`.arb` Files**:
    *   These are Application Resource Bundle files, typically containing JSON-like key-value pairs.
    *   In this export, they correspond to structured data files like `_book.yaml` (navigation) or `_footer.yaml`.
    *   **The script currently skips these files.** They are considered "skeleton" or configuration data rather than primary content, and their format requires specialized parsing to be useful.

*   **Other `.html` Files**:
    *   Generic HTML files are copied over if found, preserving their names.

## Usage

To rebuild the `consolidated` directory with the latest translations:

```bash
python3 i18n/scripts/consolidate_translations.py
```
