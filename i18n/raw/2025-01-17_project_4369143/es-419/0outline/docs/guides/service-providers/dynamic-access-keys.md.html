A partir de la versión 1.8.0, el cliente de Outline ofrece compatibilidad en versión beta con claves de acceso dinámicas. Anteriormente, la información necesaria para acceder a una VPN de Outline se almacenaba (de manera estática) en la clave de acceso. Sin embargo, con las claves de acceso dinámicas, puedes alojar esa información en una ubicación remota y actualizarla dinámicamente.

Esto significa que puedes cambiar sobre la marcha la ubicación de tu VPN, el puerto en el que se aloja, la contraseña para acceder a ella y el método de encriptación que se usa, todo sin volver a emitir claves de acceso para todos tus usuarios clientes.

Aloja tu información de acceso como un vínculo ss://

A partir de la versión 1.8.1 del cliente de Outline, la información de acceso almacenada en el vínculo de clave de acceso dinámica puede ser simplemente un vínculo ss://. Antes, la clave dinámica solo podía ser un objeto JSON como se describe abajo. Esta solución es adecuada para aquellas personas que no planean cambiar dinámicamente el servidor, el puerto o el método de encriptación, pero que de cualquier manera quieren poder cambiar el servidor al que están conectadas sobre la marcha. Este cambio elimina la necesidad de traducir la clave ss a un objeto JSON y tiene la ventaja de que la clave dinámica se puede almacenar en Hojas de cálculo de Google, una opción resistente a la censura en lugares donde se puede acceder al servicio.

  • Cómo almacenar la información de acceso en Hojas de cálculo de Google
    • Crea una Cuenta de Google.
    • Inicia una nueva hoja de cálculo a través de https://sheets.new.
    • Pega el vínculo ss:// en la celda A1.
    • Selecciona Archivo, Compartir y Publicar en la Web.
    • En la pestaña Vincular, selecciona Hoja1 y Valores separados por comas (.csv).
    • Haz clic en Publicar.
    • El vínculo se verá similar a esto: https://docs.google.com/spreadsheets/d/e/2PACX-1vTN0nxRcUf52R6fq3JdVfNqIPr2GQU2wBGsG-xI4WTaYrliCElxp0ZAAjy0PDlf_-qvCT3pnSB2jVEo/pub?gid=0&single=true&output=csv.
    • Reemplaza https:// por ssconf:// para crear la clave de acceso dinámica, que se verá de esta forma: ssconf://docs.google.com/spreadsheets/d/e/2PACX-1vTN0nxRcUf52R6fq3JdVfNqIPr2GQU2wBGsG-xI4WTaYrliCElxp0ZAAjy0PDlf_-qvCT3pnSB2jVEo/pub?gid=0&single=true&output=csv.
  • Sigue leyendo y conoce más opciones resistentes a la censura para almacenar la información de acceso.

Aloja tu información de acceso como un objeto JSON

La información de acceso también se puede almacenar como un objeto JSON que contenga las siguientes claves:

  • server: Es el dominio o la dirección IP del servidor en el que se aloja tu VPN, con comillas dobles.

  • server_port: Es el puerto en el cual se ejecuta tu servicio de VPN.

    De forma predeterminada, Outline Manager aloja tu VPN en un puerto efímero para no interferir en otros elementos que puedan estar ejecutándose en tu servidor. Si puedes, considera alojar tu VPN en un puerto estándar (como el 443), de manera que un escáner de puertos no pueda detectarlo fácilmente.

  • password: Es la frase o token secreto que se requiere para ingresar a la VPN, con comillas dobles. Por ejemplo, "¡Ábrete, sésamo!".

    Recuerda que, con las contraseñas, mientras mayor sea la entropía, mejor. Descifrar a través de fuerza bruta una contraseña que tenga veinte caracteres o más lleva, en promedio, toda la existencia del universo.

  • method: Es el método de encriptación que usa la VPN, con comillas dobles. Puedes obtener más información sobre los métodos que admitimos en el sitio web de Shadowsocks.

Ahora mismo, depende de ti como administrador de VPN elegir una ubicación para almacenar tu información de acceso que no está bloqueada. Estos son algunos ejemplos de opciones resistentes a la censura entre las que elegir:

  • pad.riseup.net

    • Esta opción no requiere que crees una cuenta nueva.
    • Después de crear tu archivo, haz clic en el botón Compartir en la parte superior derecha y activa el botón "Solo lectura". Copia el vínculo, que se verá similar a este: https://pad.riseup.net/p/r.9b3d9974b0fd86132c872eb8e5c84c24.
    • Agrega /export/txt al final del vínculo, para que se vea similar a este: https://pad.riseup.net/p/r.9b3d9974b0fd86132c872eb8e5c84c24/export/txt.
    • Reemplaza https:// por ssconf:// para crear la clave de acceso dinámica, que se verá de esta forma: ssconf://pad.riseup.net/p/r.9b3d9974b0fd86132c872eb8e5c84c24/export/txt.

  • S3 (asegúrate de usar el acceso con estilo de ruta)

    • Crea una cuenta de AWS y un bucket nuevo.
    • En la configuración del bucket "Permissions" ("Permisos"), para Object Ownership (Propiedad del objeto), selecciona "ACLs enabled" ("ACL activadas") y "Bucket owner preferred" ("Propietario del bucket preferido").
    • En Access control list (Lista de control de acceso) (LCA), asigna acceso de lectura ("Read") a Bucket ACL (ACL del bucket) para "Everyone (public access)" ("Todos [acceso público]").
    • Agrega el archivo JSON al bucket (sugerencia: créalo en un editor de texto y, luego, guárdalo con una extensión .json).
    • Verifica que el archivo se puede ver. Para ello, selecciona el objeto, elige "Copy URL" ("Copiar URL") y pruébala en una ventana del navegador incógnita. La URL debería verse similar a esto: https://my-bucket-name.s3.us-east-2.amazonaws.com/my_online_config_file.json.
    • Reemplaza la sección https:// por ssconf:// para crear la clave de acceso dinámica, que se ve similar a esto: ssconf://my-bucket-name.s3.us-east-2.amazonaws.com/my_online_config_file.json.

  • Netlify

Implementa tu propia solución de clave dinámica

Si eliges alojar la clave dinámica por tu cuenta, deberás asegurarte de habilitar las solicitudes de origen cruzado (también conocidas como "CORS"). Esto se puede hacer agregando el encabezado Access-Control-Allow-Origin a todas las respuestas del servidor con un valor comodín (p. ej., "*"), que le indica al cliente que cualquiera puede acceder a tu clave dinámica.

Obtén tu información de acceso a partir de una clave de acceso estática

Si apenas estás comenzando y quieres agregar una capa adicional de indirección a tu VPN de Outline existente en lugar de alguna implementación sofisticada personalizada, estás en el lugar correcto. Puedes extraer la información de acceso de una clave de acceso existente. En el ejemplo de abajo, se muestra dónde puedes encontrar cada uno de los datos pertinentes: "server", "server_port", "password" y "method".

Ejemplo de clave de acceso estática:

ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

  1. Server

    La ubicación "server" es la parte de la clave de acceso estática entre el símbolo "@" y el signo ":" siguiente.

    ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

  2. Server Port

    La ubicación "server_port" es la parte de la clave de acceso estática entre el segundo signo ":" y el siguiente "/".

    ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

  3. Password

    Aquí empiezan a dificultarse las cosas. La parte de la clave entre "ss://" y el símbolo "@" se codificó en un formato base64. Puedes usar un decodificador base64 para revelar lo que está oculto en esta parte:

    ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

    En nuestro caso, cuando decodificamos esta parte de la clave, obtenemos el resultado que aparece abajo, en el que la parte en negrita (después de ":") es la "password":

    chacha20-ietf-poly1305:5REfxTjltz2L

  4. Method

    Y, como quizá suponías, la parte del mismo resultado de arriba antes de ":" es el "method":

    chacha20-ietf-poly1305:5REfxTjltz2L

Crea tu clave de acceso dinámica

Cuando tienes la URL en la que se almacena tu información de acceso, todo lo que necesitas hacer para crear tu clave de acceso dinámica es reemplazar el protocolo https:// por el ssconf://. Esto le indica al cliente de Outline cómo debería manejar esta clave de acceso particular.

ANTES (protocolo https) https://s3.amazonaws.com/a-cool-s3-bucket/my-access-key.json

DESPUÉS (clave de acceso dinámica con protocolo ssconf) ssconf://s3.amazonaws.com/a-cool-s3-bucket/my-access-key.json

Esta clave se puede agregar al cliente de Outline.
