Before you use Outline, it can be helpful to review the core concepts.

How Outline Works

  1. Outline Manager: You create an Outline server, either on a cloud provider or your own hardware, and then generate access keys.
  2. Distribute keys: You share these keys with friends and family.
  3. Tunnel traffic: Users install the Outline client and use the keys to establish a secure tunnel, bypassing censorship.

Access Keys

Access keys are the credentials that allow users to connect to your Outline VPN server. They contain the necessary information for the Outline Client to establish a secure connection. There are two types of access keys: static and dynamic.

Static Access Keys

In a static access key, the server access information is encoded directly in the access key. This prevents the access information from being modified. Users import this key into the Outline Client to connect to the server.

  • Example: ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

Dynamic Access Keys

A dynamic access key encodes the location of the server access information, allowing you to decouple user access from specific server details. This means you can change your server's IP address, port, security settings, and more without needing to update the access keys you've already distributed.

See the detailed documentation on Dynamic Access Keys for more information.
