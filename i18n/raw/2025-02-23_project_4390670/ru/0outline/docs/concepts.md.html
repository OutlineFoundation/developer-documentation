Прежде чем начать использовать Outline, ознакомьтесь с основными понятиями.

Принцип работы Outline

  1. Менеджер Outline: вы создаете сервер Outline через поставщика облачных сервисов или с помощью своего оборудования, а затем генерируете ключи доступа.
  2. Распределение ключей: вы делитесь ключами доступа с друзьями и родными.
  3. Туннелирование трафика: пользователи устанавливают клиент Outline и используют ключи для создания безопасного соединения и обхода ограничений.

Ключи доступа

Ключи доступа – это учетные данные, которые используются для подключения к вашему VPN-серверу Outline. Они содержат информацию, необходимую клиенту Outline для создания защищенного подключения. Outline использует два типа ключей доступа: статические и динамические.

Статические ключи доступа

В случае статических ключей вся информация, необходимая для подключения к серверу, хранится в самом ключе, поэтому эти данные нельзя изменить. Пользователи импортируют такой ключ в клиент Outline, чтобы подключиться к серверу.

  • Пример: ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

Динамические ключи доступа

Динамический ключ доступа кодирует местоположение информации о доступе к серверу. Это позволяет отделить доступ пользователей к ключу от параметров конкретного сервера. В результате вы можете изменять IP-адрес сервера, порт, настройки безопасности и другие параметры без необходимости менять ключи, которые вы уже передали пользователям.
