As of version 1.8.0, the Outline Client has beta support for Dynamic Access Keys. Previously, the information needed to access an Outline VPN was (statically) stored in the access key itself. However with Dynamic Access Keys, you can host that information at a remote location and update it dynamically.

This means you can switch up the location of your VPN, the port it's hosted on, the password to access it, and the encryption method being used on the fly — all without having to re-issue access keys to all your client users!

Hosting your Access Information as an ss:// Link

As of Outline Client version 1.8.1, the access information stored at the dynamic access key link can simply be an ss:// link. (Previously, the dynamic key could only be a JSON object as described below.) This solution is suitable for those who do not plan to dynamically change the server, port, or encryption method, but still want to be able to change out the server being connected to on the fly. It removes the need to translate the ss key to a JSON object, and has the advantage that the dynamic key can be stored in Google Sheets, a censorship-resistant option in places where Google Sheets is accessible.

  • Storing the access information in Google Sheets
    • Create a Google account
    • Start a new spreadsheet via https://sheets.new
    • Paste the ss:// link into cell A1
    • Select File / Share / Publish to web
    • On the Link tab, select Sheet1 and Comma-separated values (.csv)
    • Click Publish
    • The link will look something like this: https://docs.google.com/spreadsheets/d/e/2PACX-1vTN0nxRcUf52R6fq3JdVfNqIPr2GQU2wBGsG-xI4WTaYrliCElxp0ZAAjy0PDlf_-qvCT3pnSB2jVEo/pub?gid=0&single=true&output=csv
    • Replace https:// with ssconf:// to create the dynamic access key, which looks something like this: ssconf://docs.google.com/spreadsheets/d/e/2PACX-1vTN0nxRcUf52R6fq3JdVfNqIPr2GQU2wBGsG-xI4WTaYrliCElxp0ZAAjy0PDlf_-qvCT3pnSB2jVEo/pub?gid=0&single=true&output=csv
  • Other censorship-resistant options for storing the access information are found below.

Hosting your Access Information as a JSON object

The access information can alternatively be stored as a JSON object that contains the following keys:

  • server: The domain or IP address of the server at which your VPN is hosted, in double quotes.

  • server_port: The port on which your VPN service is running.

    By default, the Outline Manager hosts your VPN at an ephemeral port to avoid interfering with other things that may be running on your server. If you can, consider hosting your VPN at a standard port (like 443), so it can't be easily detected by a port scanner.

  • password: The secret phrase or token required to enter the VPN, in double quotes. Think "Open Sesame!"

    Remember, with passwords, the higher the entropy, the better. A password that is twenty characters or longer takes, on average, the entire lifespan of the universe to brute force.

  • method: The encryption method used by the VPN, in double quotes. You can read more about the methods we support on the shadowsocks website.

  • pad.riseup.net

    • This option does not require creating a new account
    • After creating your file, click the Share button at the top right and turn the "Read only" toggle on. Copy the link, which will look something like this: https://pad.riseup.net/p/r.9b3d9974b0fd86132c872eb8e5c84c24
    • Add /export/txt to the end of the link, so the link looks something like this: https://pad.riseup.net/p/r.9b3d9974b0fd86132c872eb8e5c84c24/export/txt
    • Replace https:// with ssconf:// to create the dynamic access key, which looks something like this: ssconf://pad.riseup.net/p/r.9b3d9974b0fd86132c872eb8e5c84c24/export/txt

  • S3 (Make sure to use path style access)

    • Create an AWS account and create a new bucket
    • In the bucket "Permissions" settings, for Object Ownership, select "ACLs enabled" and "Bucket owner preferred"
    • In the Access control list (ACL), assign "Read" access to the Bucket ACL for "Everyone (public access)"
    • Add the JSON file to the bucket (tip: create it in a text editor and then save it with a .json extension)
    • Check that the file is viewable by selecting the object, choosing "Copy URL", and testing it in an incognito browser window. The URL should look something like this: https://my-bucket-name.s3.us-east-2.amazonaws.com/my_online_config_file.json
    • Replace the https:// with ssconf:// to create the dynamic access key, which looks something like this: ssconf://my-bucket-name.s3.us-east-2.amazonaws.com/my_online_config_file.json

Rolling your own Dynamic Key solution

Should you choose to host your dynamic key yourself, you'll need to make sure you enable Cross-Origin requests (known also as "CORS"). This can be done by adding the Access-Control-Allow-Origin header to all server responses with a wildcard value (e.g. "*"), telling the client that anyone is allowed to access your dynamic key.

Getting your Access Information from the Static Access Key

If you're just getting started and want to add an extra layer of indirection to your existing Outline VPN rather than some fancy custom deployment, don't fear! You can extract the access information from an existing access key. The example below shows you where you can find each of the relevant pieces of information: "server", "server_port", "password", and "method".

Example Static Access Key:

ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

  1. Server

    The "server" location is the part of the static access key between the "@" symbol and the following ":"

    ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

  2. Server Port

    The "server_port" location is the part of the static access key between the second ":" symbol and the following "/"

    ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

  3. Password

    This is where things get a little tricky. The part of the key that's between "ss://" and the "@" is encoded in a base64 format. You can use a base64 decoder to reveal what's hidden in this part:

    ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

    In our case, when we un-encode this part of the key, we get the result below, where the bolded portion (after the ":") is the "password":

    chacha20-ietf-poly1305:5REfxTjltz2L

  4. Method

    And, as you may have guessed, the part of that same above result before the ":" is the "method":

    chacha20-ietf-poly1305:5REfxTjltz2L

Creating your Dynamic Access Key

Once you have the URL to where your access information is stored, all you need to do to create your dynamic access key is replace the https:// protocol with the ssconf:// protocol. This tells the Outline Client how it should handle this particular access key.

BEFORE (https protocol) https://s3.amazonaws.com/a-cool-s3-bucket/my-access-key.json

AFTER (dynamic access key with ssconf protocol) ssconf://s3.amazonaws.com/a-cool-s3-bucket/my-access-key.json

This key can then be added to the Outline Client!
