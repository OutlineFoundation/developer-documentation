В клиенте Outline версии 1.8.0 есть бета-поддержка динамических ключей доступа. Ранее информация, необходимая для подключения к Outline VPN, статично хранилась в самом ключе доступа. Но благодаря динамическим ключам доступа ее можно хранить удаленно и обновлять.

Это значит, что вы можете быстро менять местоположение своей сети VPN, порт, на котором она размещена, пароль для подключения и метод шифрования, не перевыпуская ключи доступа для всех пользователей клиента.

Размещение информации о доступе в качестве ссылки ss://

В клиенте Outline версии 1.8.1 разместить информацию о доступе, хранящуюся по ссылке динамического ключа доступа, можно в виде ссылки ss://. Ранее динамический ключ представлял собой только объект JSON. Новое решение подойдет вам, если вы не планируете динамически менять сервер, порт или метод шифрования, но хотите иметь возможность быстро менять сервер, с которым установлено подключение. Вам не нужно будет переводить ключ ss в объект JSON, и вы сможете хранить динамический ключ в Google Таблицах, которые защищены от цензуры там, где они доступны.

  • Хранение информации о доступе в Google Таблицах
    • Создайте аккаунт Google.
    • Создайте новую таблицу, перейдя на страницу https://sheets.new.
    • Вставьте ссылку ss:// в ячейку A1.
    • Выберите "Файл > Поделиться > Опубликовать в интернете".
    • На вкладке "Ссылка" выберите параметры "Лист1" и "Файл CSV".
    • Нажмите "Опубликовать".
    • Вы получите ссылку следующего вида: https://docs.google.com/spreadsheets/d/e/2PACX-1vTN0nxRcUf52R6fq3JdVfNqIPr2GQU2wBGsG-xI4WTaYrliCElxp0ZAAjy0PDlf_-qvCT3pnSB2jVEo/pub?gid=0&single=true&output=csv
    • Замените https:// на ssconf://, чтобы создать динамический ключ доступа, который будет выглядеть следующим образом: ssconf://docs.google.com/spreadsheets/d/e/2PACX-1vTN0nxRcUf52R6fq3JdVfNqIPr2GQU2wBGsG-xI4WTaYrliCElxp0ZAAjy0PDlf_-qvCT3pnSB2jVEo/pub?gid=0&single=true&output=csv
  • Другие защищенные от цензуры способы хранения информации о доступе перечислены ниже.

Размещение информации о доступе в качестве объекта JSON

Информацию о доступе можно хранить в качестве объекта JSON, который содержит следующие ключи:

  • server: домен или IP-адрес сервера, на котором размещена ваша сеть VPN, в двойных кавычках.

  • server_port: порт, на котором запущен ваш сервис VPN.

    По умолчанию Менеджер Outline размещает VPN на динамическом порте, чтобы избежать пересечения с другими сервисами, запущенными на сервере. По возможности разместите свою сеть VPN на стандартном порте (например, 443), чтобы ее было сложнее распознать сканеру портов.

  • password: секретная фраза или токен, необходимый для подключения к VPN, в двойных кавычках, например "Сезам, откройся!".

    Чем сложнее пароль, тем он надежнее. Например, чтобы взломать пароль из 20 символов путем перебора всех возможных вариантов, потребуется время равное возрасту вселенной.

  • method: метод шифрования, используемый в VPN, в двойных кавычках. Подробнее о поддерживаемых методах…

Как администратор VPN, вы можете выбирать для хранения информации о доступе ресурс, который не заблокирован. Ниже перечислено несколько вариантов.

  • pad.riseup.net

    • Этот вариант не требует создания нового аккаунта.
    • После создания файла в правом верхнем углу нажмите кнопку "Поделиться" и установите переключатель "Только чтение". Скопируйте ссылку, которая будет выглядеть примерно так: https://pad.riseup.net/p/r.9b3d9974b0fd86132c872eb8e5c84c24
    • Добавьте /export/txt в конце ссылки, чтобы получилось так: https://pad.riseup.net/p/r.9b3d9974b0fd86132c872eb8e5c84c24/export/txt
    • Замените https:// на ssconf://. Будет создан динамический ключ доступа следующего вида: ssconf://pad.riseup.net/p/r.9b3d9974b0fd86132c872eb8e5c84c24/export/txt

  • S3 (обязательно используйте модель адресации path-style)

    • Создайте аккаунт AWS и новый сегмент.
    • В разделе Permissions (Разрешения) настроек сегмента для параметра Object Ownership (Право собственности на объект) выберите ACLs enabled (Списки контроля доступа включены) и Bucket owner preferred (Предпочтителен владелец сегмента).
    • В списке контроля доступа к сегменту установите разрешение на чтение (Read) для всех (Everyone (public access)).
    • Добавьте файл JSON в сегмент. Вы можете создать его в текстовом редакторе и сохранить с расширением .json.
    • Проверьте доступность файла. Для этого выберите объект, нажмите Copy URL (Копировать URL) и попробуйте открыть скопированную ссылку в браузере в режиме инкогнито. URL должен выглядеть следующим образом: https://my-bucket-name.s3.us-east-2.amazonaws.com/my_online_config_file.json
    • Замените https:// на ssconf://. Будет создан динамический ключ доступа следующего вида: ssconf://my-bucket-name.s3.us-east-2.amazonaws.com/my_online_config_file.json

Внедрение собственного решения для динамических ключей

Если вы хотите использовать собственное решение для размещения динамических ключей, обязательно включите междоменные запросы (CORS). Для этого добавьте во все ответы сервера заголовок Access-Control-Allow-Origin с подстановочным знаком (*). Таким образом доступ к динамическому ключу будет разрешен для всех.

Получение информации о доступе из статического ключа

Если вы просто хотите добавить ещё один вариант обхода для существующей сети VPN Outline, а не настраивать специальное развертывание, вы можете извлечь информацию из существующего ключа доступа. В примере ниже показано, где найти необходимые фрагменты информации: server, server_port, password и method.

Пример статического ключа доступа:

ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

  1. Сервер

    Фрагмент server, указывающий на сервер, расположен между символом @ и последующим двоеточием (:).

    ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

  2. Порт сервера

    Фрагмент server_port, указывающий на порт сервера, расположен между вторым двоеточием (:) и последующей косой чертой (/).

    ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

  3. Пароль

    Здесь всё немного сложнее. Эта часть ключа доступа, расположенная между ss:// и символом @, закодирована в формате Base64. Для ее расшифровки можно использовать специальный декодировщик.

    ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

    В нашем примере после дешифровки этой части ключа получается указанная ниже строка, в которой пароль – это выделенный полужирным шрифтом фрагмент после двоеточия.

    chacha20-ietf-poly1305:5REfxTjltz2L

  4. Метод шифрования

    Как вы могли догадаться, фрагмент до двоеточия из предыдущего результата декодирования и есть метод.

    chacha20-ietf-poly1305:5REfxTjltz2L

Создание динамического ключа доступа

Если у вас есть URL, в котором хранится информация о доступе, для создания динамического ключа вам нужно просто заменить протокол https:// на ssconf://. Это нужно для того, чтобы клиент Outline правильно обработал этот ключ доступа.

ДО (протокол https) https://s3.amazonaws.com/a-cool-s3-bucket/my-access-key.json

ПОСЛЕ (динамический ключ доступа с протоколом ssconf) ssconf://s3.amazonaws.com/a-cool-s3-bucket/my-access-key.json

Получившийся ключ можно добавить в клиент Outline.
