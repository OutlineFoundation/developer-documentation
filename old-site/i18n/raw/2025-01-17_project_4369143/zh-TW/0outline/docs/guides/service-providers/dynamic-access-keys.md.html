Outline 用戶端自 1.8.0 版起以 Beta 版形式支援動態存取金鑰。之前，存取 Outline VPN 所需的資訊是靜態儲存在存取金鑰中。現在透過動態存取金鑰，您可以將這些資訊託管於遠端位置，並動態更新。

也就是說，您可以隨時更改 VPN 的位置、託管通訊埠、存取密碼及加密方法，無需重新向全體用戶端使用者發放存取金鑰！

將存取資訊以 ss:// 連結形式託管

自 Outline 用戶端 1.8.1 版起，儲存在動態存取金鑰連結中的存取資訊可以直接採用 ss:// 連結格式 (在之前的版本中，動態金鑰只能是 JSON 物件，詳見下文)。如果您不打算動態調整伺服器、通訊埠或加密方法，但仍希望能隨時切換連線的伺服器，就很適合採用這個解決方案。這種方法免除了將 ss 金鑰轉換為 JSON 物件的步驟；此外，動態金鑰可以儲存在 Google 試算表中，因此在能夠存取 Google 試算表的地區，這是一種對抗審查的有效方式。

  • 將存取資訊儲存在 Google 試算表中
    • 建立 Google 帳戶
    • 透過 https://sheets.new 建立新試算表
    • 將 ss:// 連結貼入 A1 儲存格
    • 依序選取「檔案」>「共用」>「發布到網路」
    • 在「連結」分頁，選取「工作表 1」和「逗號分隔值(.csv)」
    • 點選「發布」
    • 生成連結的格式如下：https://docs.google.com/spreadsheets/d/e/2PACX-1vTN0nxRcUf52R6fq3JdVfNqIPr2GQU2wBGsG-xI4WTaYrliCElxp0ZAAjy0PDlf_-qvCT3pnSB2jVEo/pub?gid=0&single=true&output=csv
    • 將 https:// 替換為 ssconf:// 即可建立動態存取金鑰，格式如下：ssconf://docs.google.com/spreadsheets/d/e/2PACX-1vTN0nxRcUf52R6fq3JdVfNqIPr2GQU2wBGsG-xI4WTaYrliCElxp0ZAAjy0PDlf_-qvCT3pnSB2jVEo/pub?gid=0&single=true&output=csv
  • 下文還會介紹其他抗審查的存取資訊儲存方式。

將存取資訊以 JSON 物件形式託管

您也可以將存取資訊儲存為包含以下鍵值的 JSON 物件：

  • server：託管 VPN 伺服器的網域或 IP 位址，需用雙引號包夾。

  • server_port：VPN 服務運作的通訊埠。

    根據預設，Outline Manager 會將 VPN 託管於臨時通訊埠，避免與伺服器上可能運行的其他服務衝突。如果可以，建議將 VPN 託管於 443 之類的標準通訊埠，以免被通訊埠掃描工具輕易偵測到。

  • password：連線 VPN 所需的密語或權杖，需用雙引號包夾，類似於「芝麻開門！」。

    提醒您，密碼熵值越高越好。平均而言，長度 20 個字元以上的密碼，幾乎無法被暴力破解。

  • method：VPN 使用的加密方法，需用雙引號包夾。您可以前往 shadowsocks 網站，進一步瞭解我們支援的加密方法。

目前，VPN 管理員需要自行選擇一個不會遭到封鎖的位置來存放存取資訊。以下是一些適合對抗審查的存放位置選擇：

  • pad.riseup.net

    • 若選擇存放在這裡，就不需要建立新帳戶
    • 建立檔案後，點選右上方的「Share」按鈕，開啟「Read only」切換鈕。複製生成的連結，格式如下：https://pad.riseup.net/p/r.9b3d9974b0fd86132c872eb8e5c84c24
    • 在連結末端加上 /export/txt，如下所示：https://pad.riseup.net/p/r.9b3d9974b0fd86132c872eb8e5c84c24/export/txt
    • 將 https:// 替換為 ssconf:// 即可建立動態存取金鑰，格式如下：ssconf://pad.riseup.net/p/r.9b3d9974b0fd86132c872eb8e5c84c24/export/txt

  • S3 (務必使用路徑樣式存取格式)

    • 建立 AWS 帳戶和新 bucket
    • 在 bucket 的「Permissions」設定中，針對「Object Ownership」選取「ACLs enabled」和「Bucket owner preferred」
    • 在存取控管清單 (ACL) 中，針對「Everyone (public access)」指派 bucket ACL 的「Read」權限。
    • 將 JSON 檔案加入 bucket (提示：在文字編輯器中建立檔案，然後以 .json 副檔名儲存)
    • 檢查是否能查看該檔案，方法是選取該物件、點選「Copy URL」，然後在瀏覽器無痕視窗中嘗試開啟網址。網址格式應如下：https://my-bucket-name.s3.us-east-2.amazonaws.com/my_online_config_file.json
    • 將 https:// 替換為 ssconf:// 即可建立動態存取金鑰，格式如下：ssconf://my-bucket-name.s3.us-east-2.amazonaws.com/my_online_config_file.json

  • Netlify

自建動態金鑰解決方案

如果您選擇自行託管動態金鑰，請務必啟用跨來源要求 (也稱為「CORS」)。您可以將 Access-Control-Allow-Origin 標頭加入所有伺服器回應，並將其值設為萬用字元 (例如「*」)，告訴用戶端任何人都能存取您的動態金鑰。

從靜態存取金鑰取得存取資訊

如果您剛開始使用，想在現有的 Outline VPN 中加入額外中介層，而非採用複雜的自訂部署配置，別擔心！您可以從現有的存取金鑰中擷取存取資訊。下方示例介紹了「server」、「server_port」、「password」和「method」分別位於何處。

靜態存取金鑰示例：

ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

  1. 伺服器

    在靜態存取金鑰中，「server」是「@」符號與後方「:」之間的部分

    ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

  2. 伺服器通訊埠

    在靜態存取金鑰中，「server_port」是第二個「:」符號與後方「/」之間的部分

    ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

  3. 密碼

    這部分稍微複雜。金鑰中介於「ss://」和「@」之間的部分是以 base64 格式編碼。您可以使用 base64 解碼器解析此部分隱藏的內容：

    ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

    在我們的例子中，解碼後的結果如下，粗體部分 (「:」後面) 即為「password」：

    chacha20-ietf-poly1305:5REfxTjltz2L

  4. 方法

    您可能已猜到，以上結果中「:」前的部分就是「method」：

    chacha20-ietf-poly1305:5REfxTjltz2L
