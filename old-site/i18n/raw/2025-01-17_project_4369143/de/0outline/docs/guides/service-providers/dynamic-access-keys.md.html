Die Version 1.8.0 des Outline-Client bietet einen Betasupport für dynamische Zugriffsschlüssel. Bisher waren die Informationen, die für den Zugriff auf ein Outline-VPN nötig waren, (statisch) in dem Zugriffsschlüssel gespeichert. Mit dynamischen Zugriffsschlüsseln können Sie diese Informationen an einem entfernten Standort hosten und dynamisch aktualisieren.

Das heißt, Sie können den Standort Ihres VPN, den Port, auf den es gehostet wurde, das Passwort, mit dem Sie darauf zugreifen können, und die verwendete Verschlüsselungsmethode spontan ändern. Dabei müssen Sie Ihren Clientnutzern noch nicht einmal den Zugriffsschlüssel neu bereitstellen.

Die Zugriffsinformationen als ss://-Link hosten

Bei Version 1.8.1 des Outline-Client kann die Zugriffsinformation, die in dem Link des dynamischen Zugriffsschlüssels gespeichert ist, einfach ein ss://-Link sein. (Bisher konnte der dynamische Zugriffsschlüssel nur ein JSON-Objekt sein wie weiter unten beschrieben.) Diese Lösung eignet sich für alle, die ihren Server, Port oder ihre Verschlüsselungsmethode nicht dynamisch ändern, aber trotzdem in der Lage sein wollen, den mit ihnen verbundenen Server spontan zu ändern. Dadurch ist es nicht mehr erforderlich, den ss-Schlüssel in ein JSON-Objekt umzuwandeln. Die Lösung hat den Vorteil, dass der dynamische Schlüssel in Google Sheets gespeichert werden kann. Diese zensurresistente Option ist überall verfügbar, wo der Zugriff auf Google Sheets möglich ist.

  • Zugriffsinformationen in Google Sheets speichern
    • Erstellen Sie ein Google-Konto.
    • Öffnen Sie über https://sheets.new eine neue Tabelle.
    • Fügen Sie in Zelle A1 den ss://-Link ein.
    • Wählen Sie „Datei“ aus, und dann „Freigeben“ und „Im Web veröffentlichen“.
    • Wählen sie unter „Link“ „Tabelle1“ und „Kommagetrennte Werte (CSV)“ aus.
    • Klicken Sie auf „Veröffentlichen“.
    • Der Link wird in etwa so aussehen: https://docs.google.com/spreadsheets/d/e/2PACX-1vTN0nxRcUf52R6fq3JdVfNqIPr2GQU2wBGsG-xI4WTaYrliCElxp0ZAAjy0PDlf_-qvCT3pnSB2jVEo/pub?gid=0&single=true&output=csv.
    • Ersetzen Sie https:// durch ssconf://, um einen dynamischen Zugriffsschlüssel zu erstellen. Dieser sieht in etwa so aus: ssconf://docs.google.com/spreadsheets/d/e/2PACX-1vTN0nxRcUf52R6fq3JdVfNqIPr2GQU2wBGsG-xI4WTaYrliCElxp0ZAAjy0PDlf_-qvCT3pnSB2jVEo/pub?gid=0&single=true&output=csv.
  • Weitere Optionen, mit denen Sie Zensierungen umgehen und die Zugriffsinformationen speichern können, finden Sie weiter unten.

Die Zugriffsinformationen als JSON-Objekt hosten

Die Zugriffsinformationen können alternativ als JSON-Objekt gespeichert werden, das die folgenden Schlüssel enthält:

  • server: die Domain oder IP-Adresse des Servers, auf dem Ihr VPN gehostet wird, in doppelten Anführungszeichen.

  • server_port: der Port, auf dem Ihr VPN-Dienst ausgeführt wird.

    Der Outline-Manager hostet Ihr VPN standardmäßig auf einem sitzungsspezifischen Port. Dadurch werden Überlagerungen mit anderen Programmen oder Diensten vermieden, die möglicherweise auf Ihrem Server ausgeführt werden. Falls möglich, ziehen Sie das Hosten Ihres VPN auf einem Standard-Port (wie 443) in Betracht. Damit kann es nicht so einfach von einem Port-Scanner entdeckt werden.

  • password: die Passphrase bzw. das Token, die bzw. das für den Zugriff auf das VPN nötig ist, in doppelten Anführungszeichen. Zum Beispiel "Sesam öffne dich!"

    Aber denken Sie daran: je unvorhersehbarer das Passwort, umso besser. Ein Passwort mit zwanzig oder mehr Zeichen bietet für gewöhnlich einen ausreichenden Schutz gegen die Brute-Force-Methode.

  • method: die Verschlüsselungsmethode, die vom VPN genutzt wird, in doppelten Anführungszeichen. Weitere Informationen zu den von uns unterstützten Methoden erhalten Sie auf der Shadowsocks-Website.

JSON-Beispiel:

HINWEIS: Nutzen Sie unbedingt das Zeichen für gerade doppelte Anführungszeichen: ". Viele Plattformen verwenden schräge Anführungszeichen wie dieses: “ – vermeiden Sie diese Art von Anführungszeichen. Sie können nicht geparst werden.

Jetzt können Sie als VPN-Verwalter einen Standort auswählen, an dem Sie Ihre Zugriffsinformationen speichern und der nicht blockiert ist. Im Folgenden zeigen wir Ihnen mehrere Beispiele für zensurresistente Optionen:

  • pad.riseup.net

    • Bei dieser Option müssen Sie kein neues Konto einrichten.
    • Nachdem Sie Ihre Datei erstellt haben, klicken Sie rechts oben auf die Schaltfläche „Freigeben“ und aktivieren Sie den Schreibschutz. Kopieren Sie den Link, der in etwa so aussieht: https://pad.riseup.net/p/r.9b3d9974b0fd86132c872eb8e5c84c24
    • Fügen Sie /export/txt am Ende des Links ein. Damit erscheint der Link dann so: https://pad.riseup.net/p/r.9b3d9974b0fd86132c872eb8e5c84c24/export/txt
    • Ersetzen Sie https:// durch ssconf://, um einen dynamischen Zugriffsschlüssel zu erstellen. Dieser sieht in etwa so aus: ssconf://pad.riseup.net/p/r.9b3d9974b0fd86132c872eb8e5c84c24/export/txt

  • S3 (Verwenden Sie einen Zugriff mit Zugriffspfad)

    • Erstellen Sie ein AWS-Konto und einen neuen Bucket.
    • Wählen Sie in den Bucket-Einstellungen für Berechtigungen für die Objektinhaberschaft „ACLsaktiviert“ und „Bevorzugter Bucket-Besitzer“ aus.
    • Erteilen Sie in der Access Control List (ACL) Lesezugriff für alle bzw. den öffentlichen Zugriff auf die Bucket-ACL.
    • Fügen sie dem Bucket die JSON-Datei hinzu. (Tipp: Erstellen Sie sie in einem Texteditor und speichern Sie sie dann mit einer .json-Erweiterung.)
    • Prüfen Sie, ob die Datei sichtbar ist. Wählen Sie dazu das Objekt und dann „URL kopieren“ aus. Testen Sie sie in einem Inkognitofenster im Browser. Die URL sollte in etwa so aussehen: https://my-bucket-name.s3.us-east-2.amazonaws.com/my_online_config_file.json
    • Ersetzen Sie „https://“ mit „ssconf://“. So erstellen Sie einen dynamischen Zugriffsschlüssel, der in etwa so aussieht: ssconf://my-bucket-name.s3.us-east-2.amazonaws.com/my_online_config_file.json

  • Netlify

Eine eigene Lösung für dynamische Schlüssel anwenden

Wenn Sie Ihren dynamischen Schlüssel selbst hosten möchten, müssen Sie ursprungsübergreifende Anfragen (auch bekannt als „CORS“) aktivieren. Dafür fügen Sie allen Serverantworten mit einem Platzhalterwert (z. B. „*“) den Access-Control-Allow-Origin-Header hinzu und teilen dem Client so mit, dass jeder auf Ihren dynamischen Schlüssel zugreifen kann.

Die Zugriffsinformationen vom statischen Zugriffsschlüssel erhalten

Wenn Sie noch nicht so gut in der Materie stecken und Ihrem bestehenden Outline-VPN lieber eine zusätzliche Ebene der Indirektion hinzufügen möchten statt einer benutzerdefinierten Bereitstellung: keine Sorge! Sie können die Zugriffsinformationen aus einem bestehenden Zugriffsschlüssel extrahieren. Das Beispiel weiter unten veranschaulicht, wo Sie jede einzelne dieser wichtigen Informationen finden können: „server“, „server_port“, „password“ und „method“.

Beispiel statischer Zugriffsschlüssel:

ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

  1. Server

    „server“ ist der Teil des statischen Zugriffsschlüssels zwischen dem „@“-Symbol und dem nachfolgenden „:“.

    ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

  2. Server Port

    „server_port“ ist der Teil des statischen Zugriffsschlüssels zwischen dem zweiten „:“-Symbol und dem nachfolgenden „/“.

    ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

  3. Passwort

    An dieser Stelle wird es ein bisschen knifflig. Der Teil des Schlüssels zwischen „ss://“ und dem „@“ ist Base64-codiert. Sie können einen Base64-Decoder nutzen, um zu sehen, was in diesem Teil ausgeblendet ist:

    ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

    Wenn wir diesen Teil des Schlüssels decodieren, erhalten wir das folgende Ergebnis. Der fettgedruckte Abschnitt (nach dem „:“) ist das „password“:

    chacha20-ietf-poly1305:5REfxTjltz2L

  4. Method

    Wie Sie vielleicht schon erraten haben, ist der Teil des obigen Ergebnisses vor dem „:“ die „method“:

    chacha20-ietf-poly1305:5REfxTjltz2L

Einen eigenen dynamischen Zugriffsschlüssel erstellen

Sobald Sie die URL haben, unter der Ihre Zugriffsinformationen gespeichert sind, müssen Sie nur noch das https://-Protokoll mit dem ssconf://-Protokoll ersetzen. Damit ist Ihr dynamischer Zugriffsschlüssel erstellt. Mit dem Protokoll sagen Sie dem Outline-Client, wie er mit diesem speziellen Zugriffsschlüssel verfahren soll.

DAVOR (https-Protokoll) https://s3.amazonaws.com/a-cool-s3-bucket/my-access-key.json

DANACH (dynamischer Zugriffsschlüssel mit ssconf-Protokoll) ssconf://s3.amazonaws.com/a-cool-s3-bucket/my-access-key.json

Dieser Schlüssel kann jetzt im Outline-Client hinzugefügt werden.
