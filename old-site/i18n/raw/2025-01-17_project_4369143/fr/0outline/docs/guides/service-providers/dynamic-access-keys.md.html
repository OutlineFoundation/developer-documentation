À partir de la version 1.8.0, le client Outline prend les clés d'accès dynamiques en charge en version bêta. Auparavant, les informations nécessaires pour accéder à un VPN Outline étaient stockées (de façon statique) dans la clé d'accès elle-même. Grâce aux clés d'accès dynamiques, vous pouvez désormais héberger ces informations sur un emplacement distant et les mettre à jour de manière dynamique.

Vous pouvez ainsi modifier à la volée l'emplacement de votre VPN, le port sur lequel il est hébergé, le mot de passe pour y accéder et la méthode de chiffrement utilisée, sans avoir à regénérer des clés d'accès pour tous les utilisateurs du client.

Héberger vos informations d'accès sous forme de lien ss://

À partir de la version 1.8.1 du client Outline, les informations d'accès stockées à l'adresse du lien de la clé d'accès dynamique peuvent simplement être hébergées sous forme de lien ss://. Dans les versions précédentes, la clé dynamique ne pouvait être qu'un objet JSON, comme décrit ci-dessous. Cette solution convient aux utilisateurs qui ne prévoient pas de modifier de manière dynamique le serveur, le port ou la méthode de chiffrement, mais qui souhaitent quand même pouvoir modifier à la volée le serveur auquel ils se connectent. Elle évite d'avoir à convertir la clé ss en objet JSON et offre la possibilité de stocker la clé dynamique dans Google Sheets, ce qui permet de résister à la censure dans les zones où Google Sheets est accessible.

  • Stocker les informations d'accès dans Google Sheets
    • Créez un compte Google.
    • Créez une feuille de calcul sur https://sheets.new.
    • Collez le lien ss:// dans la cellule A1.
    • Sélectionnez Fichier > Partager > Publier sur le Web.
    • Dans l'onglet "Lien", sélectionnez "Feuille 1" et "Valeurs séparées par des virgules (.csv)".
    • Cliquez sur "Publier".
    • Le lien doit ressembler à ceci : https://docs.google.com/spreadsheets/d/e/2PACX-1vTN0nxRcUf52R6fq3JdVfNqIPr2GQU2wBGsG-xI4WTaYrliCElxp0ZAAjy0PDlf_-qvCT3pnSB2jVEo/pub?gid=0&single=true&output=csv
    • Remplacez https:// par ssconf:// pour créer la clé d'accès dynamique, qui doit ressembler à ceci : ssconf://docs.google.com/spreadsheets/d/e/2PACX-1vTN0nxRcUf52R6fq3JdVfNqIPr2GQU2wBGsG-xI4WTaYrliCElxp0ZAAjy0PDlf_-qvCT3pnSB2jVEo/pub?gid=0&single=true&output=csv
  • D'autres options de stockage des informations d'accès résistant à la censure sont présentées ci-dessous.

Héberger vos informations d'accès sous forme d'objet JSON

Les informations d'accès peuvent également être stockées sous la forme d'un objet JSON contenant les clés suivantes :

  • server : domaine ou adresse IP du serveur sur lequel votre VPN est hébergé, entre guillemets.

  • server_port : port sur lequel votre service VPN s'exécute.

    Par défaut, Outline Manager héberge votre VPN sur un port éphémère pour éviter toute interférence avec d'autres éléments qui pourraient s'exécuter sur votre serveur. Si possible, envisagez d'héberger votre VPN sur un port standard (443, par exemple) afin qu'il ne soit pas facilement détectable par un scanner de ports.

  • password : code secret ou jeton nécessaire pour entrer dans le VPN, entre guillemets. Par exemple, "Sésame ouvre-toi".

    N'oubliez pas que plus l'entropie d'un mot de passe est élevée, mieux c'est. Il faut en moyenne toute la durée de vie de l'univers pour craquer un mot de passe de 20 caractères ou plus par force brute.

  • method : méthode de chiffrement utilisée par le VPN, entre guillemets. Pour connaître les méthodes acceptées, consultez le site Web de Shadowsocks.

En tant qu'administrateur du VPN, c'est à vous de choisir un emplacement qui n'est pas bloqué où stocker vos informations d'accès. Voici quelques exemples d'options résistant à la censure :

  • pad.riseup.net

    • Vous n'avez pas besoin de créer un compte avec cette option.
    • Une fois votre fichier créé, cliquez sur le bouton "Share" (Partager) en haut à droite et activez l'option "Read only" (Lecture seule). Copiez le lien, qui doit ressembler à ceci : https://pad.riseup.net/p/r.9b3d9974b0fd86132c872eb8e5c84c24
    • Ajoutez "/export/txt" à la fin du lien, de sorte qu'il ressemble à ceci : https://pad.riseup.net/p/r.9b3d9974b0fd86132c872eb8e5c84c24/export/txt
    • Remplacez https:// par ssconf:// pour créer la clé d'accès dynamique, qui doit ressembler à ceci : ssconf://pad.riseup.net/p/r.9b3d9974b0fd86132c872eb8e5c84c24/export/txt

  • S3 (Veillez à utiliser un accès de type chemin d'accès)

    • Créez un compte AWS, puis un bucket.
    • Dans les paramètres "Permissions" (Autorisations) du bucket, pour la propriété des objets, sélectionnez "ACLs enabled" (LCA activées) et "Bucket owner preferred" (Propriétaire du bucket préféré).
    • Dans la liste de contrôle d'accès (LCA), attribuez l'accès "Read" (Lecture) à la LCA du bucket pour "Everyone (public access)" (Tout le monde [accès public]).
    • Ajoutez le fichier JSON au bucket. Conseil : créez le fichier dans un éditeur de texte, puis enregistrez-le avec l'extension .json.
    • Vérifiez que le fichier est consultable en sélectionnant l'objet. Cliquez ensuite sur "Copy URL" (Copier l'URL), puis testez le lien dans une fenêtre de navigation privée. L'URL doit ressembler à ceci : https://my-bucket-name.s3.us-east-2.amazonaws.com/my_online_config_file.json
    • Remplacez "https://" par "ssconf://" pour créer la clé d'accès dynamique, qui doit ressembler à ceci : ssconf://my-bucket-name.s3.us-east-2.amazonaws.com/my_online_config_file.json

Déployer votre propre solution de clé dynamique

Si vous choisissez d'héberger vous-même votre clé dynamique, vous devez veiller à activer les requêtes multi-origines (également appelées CORS). Pour cela, ajoutez l'en-tête Access-Control-Allow-Origin à toutes les réponses du serveur avec un caractère générique ("*", par exemple), afin d'indiquer au client que n'importe qui est autorisé à accéder à votre clé dynamique.

Obtenir vos informations d'accès à partir de la clé d'accès statique

Vous débutez et souhaitez ajouter un niveau d'indirection supplémentaire à votre VPN Outline existant plutôt que de créer un déploiement personnalisé plus sophistiqué ? Pas de panique ! Vous pouvez extraire les informations d'accès d'une clé d'accès existante. L'exemple ci-dessous vous montre où vous pouvez trouver les informations correspondantes pour "server", "server_port", "password" et "method".

Exemple de clé d'accès statique :

ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

  1. Serveur

    La valeur "server" correspond à la partie de la clé d'accès statique qui se trouve entre le symbole "@" et le second ":" :

    ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

  2. Port du serveur

    La valeur "server_port" correspond à la partie de la clé d'accès statique qui se trouve entre le second ":" et le "/" suivant :

    ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

  3. Mot de passe

    C'est là que les choses se compliquent légèrement. La partie de la clé entre "ss://" et le symbole "@" est encodée au format base64. Vous pouvez utiliser un décodeur base64 pour afficher ce qui est masqué dans cette partie :

    ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

    Dans notre cas, lorsque nous décodons cette partie de la clé, nous obtenons le résultat ci-dessous, où la partie en gras (après le ":") correspond à la valeur "password" :

    chacha20-ietf-poly1305:5REfxTjltz2L

  4. Méthode

    Enfin, comme vous l'avez peut-être deviné, la valeur "method" correspond à la partie du résultat ci-dessus qui se trouve avant le ":" :

    chacha20-ietf-poly1305:5REfxTjltz2L

Créer votre clé d'accès dynamique

Une fois que vous disposez de l'URL où sont stockées vos informations d'accès, il vous suffit de remplacer le protocole https:// par le protocole ssconf:// pour créer votre clé d'accès dynamique. Cela permet d'indiquer au client Outline la manière dont il doit traiter cette clé d'accès spécifique.

AVANT (protocole HTTPS) https://s3.amazonaws.com/a-cool-s3-bucket/my-access-key.json

APRÈS (clé d'accès dynamique utilisant le protocole SSCONF) ssconf://s3.amazonaws.com/a-cool-s3-bucket/my-access-key.json

Vous pouvez ensuite ajouter cette clé au client Outline.
