从 1.8.0 版开始，Outline 客户端新增了动态访问密钥 Beta 版功能。在此之前，访问 Outline VPN 所需的信息是以静态方式存储于访问密钥中。不过，借助动态访问密钥，您可以将这些信息托管在远程位置并动态更新。

这意味着，您可以即时切换 VPN 的位置、托管端口、访问密码以及加密方式，而无需重新向所有客户端用户签发访问密钥！

以 ss:// 链接的形式托管访问信息

从 Outline 客户端 1.8.1 版开始，可以直接在动态访问密钥链接中以 ss:// 链接的形式存储访问信息。之前的版本仅支持 JSON 对象形式的动态密钥，如下文所述此解决方案适合有以下需求的用户：不打算动态更改服务器、端口或加密方式，但仍希望能够即时更改当前所连接的服务器。这种解决方案省去了将 ss 密钥转换为 JSON 对象的麻烦，并且还有以下优势：动态密钥可以存储在 Google 表格中，针对可以访问 Google 表格的地区提供了规避审查的方法。

  • 将访问信息存储在 Google 表格中
    • 创建 Google 账号
    • 通过 https://sheets.new 新建一个电子表格
    • 将 ss:// 链接粘贴到单元格 A1 中
    • 依次选择“文件”/“共享”/“发布到网络”
    • 在“链接”标签页下，依次选择“工作表 1”和“逗号分隔值 (.csv)”
    • 点击“发布”
    • 生成的链接类似于：https://docs.google.com/spreadsheets/d/e/2PACX-1vTN0nxRcUf52R6fq3JdVfNqIPr2GQU2wBGsG-xI4WTaYrliCElxp0ZAAjy0PDlf_-qvCT3pnSB2jVEo/pub?gid=0&single=true&output=csv
    • 将 https:// 替换为 ssconf:// 以创建动态访问密钥，其形式类似于： ssconf://docs.google.com/spreadsheets/d/e/2PACX-1vTN0nxRcUf52R6fq3JdVfNqIPr2GQU2wBGsG-xI4WTaYrliCElxp0ZAAjy0PDlf_-qvCT3pnSB2jVEo/pub?gid=0&single=true&output=csv
  • 下面还介绍了一些其他能够规避审查的访问信息存储方式。

以 JSON 对象的形式托管访问信息

访问信息也可以存储为包含以下密钥的 JSON 对象：

  • server：托管 VPN 的服务器的域名或 IP 地址，用英文双引号括起。

  • server_port：运行 VPN 服务的端口。

    默认情况下，Outline 管理器将您的 VPN 托管在临时端口上，以免干扰可能在您的服务器上运行的其他服务。如果可以，建议您将 VPN 托管在标准端口上（如 443 端口），这样就不会被端口扫描程序轻易检测到。

  • password：进入 VPN 所需要的秘密口令或令牌，用英文双引号括起。类似于 "Open Sesame!"

    请注意，密码的熵越高，安全性越好。平均来说，字符数不少于 20 个的密码几乎无法暴力破解。

  • method：VPN 采用的加密方法，用英文双引号括起。您可以在 shadowsocks 网站上详细了解我们支持的加密方法。

现在，作为 VPN 管理员，您需要选择一个没有被封锁的位置来存储您的访问信息。下面是一些可供您选择的规避审查方法：

  • pad.riseup.net

    • 此方法无需创建新账号
    • 创建文件后，点击右上角的“Share”（分享）按钮，并将“Read only”（只读）开关切换为开启状态。复制链接，其形式类似于：https://pad.riseup.net/p/r.9b3d9974b0fd86132c872eb8e5c84c24
    • 在链接末尾添加 /export/txt，添加后的链接类似于：https://pad.riseup.net/p/r.9b3d9974b0fd86132c872eb8e5c84c24/export/txt
    • 将 https:// 替换为 ssconf://，以便创建动态访问密钥，其形式类似于：ssconf://pad.riseup.net/p/r.9b3d9974b0fd86132c872eb8e5c84c24/export/txt

  • S3（确保使用路径样式访问）

    • 创建一个 AWS 账号和一个新的存储桶
    • 在存储桶的“Permissions”（权限）设置中，针对“Object Ownership”（对象所有权），依次选择“ACLs enabled”（启用 ACL）和“Bucket owner preferred”（首选存储桶所有者）
    • 在访问控制列表 (ACL) 中，为“Everyone (public access)”（所有人 [公开访问]）分配存储桶 ACL 的“Read”（读取）访问权限
    • 向存储桶添加 JSON 文件（提示：在文本编辑器中创建该文件，然后使用 .json 扩展名保存）
    • 检查是否可查看该文件，方法是选择相应对象，再选择“Copy URL”（复制网址），然后在浏览器的无痕式窗口中进行测试。该网址应该类似于：https://my-bucket-name.s3.us-east-2.amazonaws.com/my_online_config_file.json
    • 将 https:// 替换为 ssconf:// 以创建动态访问密钥，其形式类似于：ssconf://my-bucket-name.s3.us-east-2.amazonaws.com/my_online_config_file.json

  • Netlify

构建您自己的动态密钥解决方案

如果您选择自行托管动态密钥，则需要确保启用非同源请求，也称为“CORS”。操作方法是：使用通配符值（例如“*”）向所有服务器响应添加 Access-Control-Allow-Origin 标头，以此告诉客户端所有人都可以访问您的动态密钥。

从静态访问密钥中获取您的访问信息

如果您是新手，并不想实施复杂的自定义部署，只是想给现有 Outline VPN 额外增加一个间接层，不用担心！您可以从现有访问密钥中提取访问信息。下方示例展示了可以从哪里找到以下各段密钥的信息：“server”“server_port”“password”和“method”。

静态访问密钥示例：

ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

  1. 服务器

    “server”信息位于静态访问密钥中“@”符号与后面的“:”符号之间

    ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

  2. 服务器端口

    “server_port”信息位于静态访问密钥中第二个“:”符号与后面的“/”符号之间

    ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

  3. 密码

    此处略微复杂。密钥中“ss://”与“@”之间的部分采用 base64 格式进行编码。您可以使用 base64 解码器查看这部分中隐藏的信息：

    ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

    在示例中，我们对密钥的这部分解码后，可以得到下方所示结果，其中加粗的部分（“:”后面的内容）即为“密码”：

    chacha20-ietf-poly1305:5REfxTjltz2L

  4. 加密方法

    您可能已经猜到了，在上面的结果中，“:”之前的部分即为“加密方法”：

    chacha20-ietf-poly1305:5REfxTjltz2L

创建动态访问密钥

获取存储访问信息的网址之后，您只需将 https:// 协议替换为 ssconf:// 协议，即可创建动态访问密钥。这将指示 Outline 客户端应如何处理此特定的访问密钥。

替换前（https 协议） https://s3.amazonaws.com/a-cool-s3-bucket/my-access-key.json

替换后（采用 ssconf 协议的动态访问密钥） ssconf://s3.amazonaws.com/a-cool-s3-bucket/my-access-key.json

然后就可以将该密钥添加到 Outline 客户端！
