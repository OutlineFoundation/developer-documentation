ตั้งแต่เวอร์ชัน 1.8.0 เป็นต้นไป ไคลเอ็นต์ Outline จะมีการรองรับเบต้าสำหรับคีย์การเข้าถึงแบบไดนามิก ซึ่งก่อนหน้านี้ ข้อมูลที่จำเป็นในการเข้าถึง Outline VPN จะจัดเก็บ (แบบคงที่) ในตัวคีย์การเข้าถึงเลย อย่างไรก็ตาม เมื่อใช้คีย์การเข้าถึงแบบไดนามิก คุณสามารถโฮสต์ข้อมูลดังกล่าวในสถานที่ระยะไกลและอัปเดตแบบไดนามิกได้

การดำเนินการนี้ช่วยให้คุณสลับตำแหน่งของ VPN, พอร์ตที่โฮสต์, รหัสผ่านในการเข้าถึง และวิธีการเข้ารหัสที่ใช้งานได้อย่างรวดเร็ว ซึ่งทั้งหมดนี้ทำได้โดยไม่ต้องออกคีย์การเข้าถึงใหม่ให้กับผู้ใช้ไคลเอ็นต์ทั้งหมด

ตั้งแต่ไคลเอ็นต์ Outline เวอร์ชัน 1.8.1 เป็นต้นไป ข้อมูลการเข้าถึงที่จัดเก็บในลิงก์คีย์การเข้าถึงแบบไดนามิกอาจเป็นเพียงลิงก์ ss:// (ก่อนหน้านี้ คีย์แบบไดนามิกจะเป็นออบเจ็กต์ JSON ได้เท่านั้น ดังที่อธิบายไว้ด้านล่าง) โซลูชันนี้เหมาะสำหรับผู้ที่ไม่ได้วางแผนที่จะเปลี่ยนเซิร์ฟเวอร์ พอร์ต หรือวิธีการเข้ารหัสอยู่เสมอ แต่ยังคงต้องการเปลี่ยนเซิร์ฟเวอร์ที่เชื่อมต่ออยู่ได้อย่างรวดเร็ว โดยวิธีนี้จะช่วยขจัดความจำเป็นในการแปลคีย์ ss ไปเป็นออบเจ็กต์ JSON และมีข้อดีคือสามารถเก็บคีย์แบบไดนามิกไว้ใน Google ชีตได้ ซึ่งเป็นตัวเลือกที่ป้องกันการเซ็นเซอร์ในที่ที่สามารถเข้าถึง Google Sheets ได้

โฮสติ้งข้อมูลการเข้าถึงในรูปแบบออบเจ็กต์ JSON

ข้อมูลการเข้าถึงสามารถจัดเก็บในรูปแบบออบเจ็กต์ JSON ที่มีคีย์ต่อไปนี้

  • server: โดเมนหรือที่อยู่ IP ของเซิร์ฟเวอร์ที่โฮสต์ VPN ของคุณ โดยใส่เครื่องหมายคำพูดคู่

  • server_port: พอร์ตที่บริการ VPN ของคุณกำลังทำงานอยู่

    โดยค่าเริ่มต้น Outline Manager จะโฮสต์ VPN ของคุณที่พอร์ตชั่วคราวเพื่อหลีกเลี่ยงการรบกวนสิ่งอื่นๆ ที่อาจทำงานอยู่บนเซิร์ฟเวอร์ หากเป็นไปได้ ให้พิจารณาโฮสต์ VPN ที่พอร์ตมาตรฐาน (เช่น 443) เพื่อไม่ให้เครื่องสแกนพอร์ตตรวจพบได้ง่าย

  • password: วลีหรือโทเค็นลับที่จำเป็นต้องใช้เพื่อเข้า VPN โดยใส่เครื่องหมายคำพูดคู่ ตัวอย่างเช่น "Open Sesame!"

    โปรดทราบว่าเมื่อใช้รหัสผ่าน หากคาดเดาได้ยากเท่าใด ก็จะมีประสิทธิภาพดีเท่านั้น ซึ่งรหัสผ่านที่มีความยาวตั้งแต่ 20 อักขระขึ้นไป โดยเฉลี่ยแล้วจะต้องใช้เวลาไม่รู้จบจึงจะคาดเดาได้

  • method: วิธีการเข้ารหัสที่ VPN ใช้ โดยใส่เครื่องหมายคำพูดคู่ คุณสามารถอ่านเพิ่มเติมเกี่ยวกับวิธีการที่เราสนับสนุนได้ที่เว็บไซต์ shadowsocks

ตัวอย่าง JSON

หมายเหตุ: ตรวจสอบว่าคุณใช้อักขระคำพูดคู่ " แบบมาตรฐาน มีแพลตฟอร์มจำนวนมากเปลี่ยนอักขระนี้ให้เป็นเครื่องหมายคำพูดที่แปลกตาขึ้น เช่น -- " -- ให้หลีกเลี่ยงการใช้ในลักษณะนี้ เนื่องจากระบบจะไม่แยกวิเคราะห์

ตอนนี้ก็ขึ้นอยู่กับคุณในฐานะผู้จัดการ VPN ในการเลือกตำแหน่งที่จะจัดเก็บข้อมูลการเข้าถึงที่ไม่ถูกบล็อกของคุณ ตัวอย่างตัวเลือกการป้องกันการเซ็นเซอร์มีดังนี้

  • pad.riseup.net

    • ไม่จำเป็นต้องสร้างบัญชีใหม่สำหรับตัวเลือกนี้
    • หลังจากสร้างไฟล์แล้ว ให้คลิกปุ่ม "Share" ที่ด้านขวาบนแล้วสลับเพื่อเปิด "Read only" คัดลอกลิงก์ซึ่งมีลักษณะดังนี้ https://pad.riseup.net/p/r.9b3d9974b0fd86132c872eb8e5c84c24
    • เพิ่ม /export/txt ต่อท้ายลิงก์ดังกล่าวเพื่อให้มีลักษณะดังนี้ https://pad.riseup.net/p/r.9b3d9974b0fd86132c872eb8e5c84c24/export/txt
    • แทนที่ https:// ด้วย ssconf:// เพื่อสร้างคีย์การเข้าถึงแบบไดนามิกซึ่งจะมีลักษณะดังนี้ ssconf://pad.riseup.net/p/r.9b3d9974b0fd86132c872eb8e5c84c24/export/txt

    • สร้างบัญชี AWS และสร้างที่เก็บข้อมูลใหม่
    • ในการตั้งค่า "Permissions" ของที่เก็บข้อมูล ให้เลือก "ACLs enabled" และ "Bucket owner preferred" ในส่วน Object Ownership
    • ในรายการควบคุมการเข้าถึง (ACL) ให้มอบหมายสิทธิ์ "Read" ใน ACL ของที่เก็บข้อมูลสำหรับ "Everyone (public access)"
    • เพิ่มไฟล์ JSON ในที่เก็บข้อมูล (เคล็ดลับ: สร้างไฟล์ในเครื่องมือแก้ไขข้อความจากนั้นบันทึกด้วยนามสกุล .json)
    • ตรวจสอบว่าไฟล์สามารถดูได้โดยเลือกออบเจ็กต์ แล้วเลือก "คัดลอก URL" จากนั้นทดสอบในหน้าต่างเบราว์เซอร์ที่ไม่ระบุตัวตน URL ควรมีลักษณะดังนี้ https://my-bucket-name.s3.us-east-2.amazonaws.com/my_online_config_file.json

การใช้งานโซลูชันคีย์แบบไดนามิกของคุณเอง

หากเลือกที่จะโฮสต์คีย์แบบไดนามิกด้วยตนเอง คุณต้องตรวจสอบว่าได้เปิดใช้งานคำขอข้ามต้นทาง (เรียกอีกอย่างว่า "CORS") แล้ว ซึ่งสามารถทำได้โดยเพิ่มส่วนหัว Access-Control-Allow-Origin ในการตอบกลับทั้งหมดของเซิร์ฟเวอร์ที่มีค่าไวลด์การ์ด (เช่น "*") เพื่อเป็นการบอกไคลเอ็นต์ว่าทุกคนได้รับอนุญาตให้เข้าถึงคีย์แบบไดนามิกของคุณ

การรับข้อมูลการเข้าถึงจากคีย์การเข้าถึงแบบคงที่

หากคุณเพิ่งเริ่มต้นใช้งานและต้องการเพิ่มการอ้างอิงโดยอ้อมอีกชั้นให้กับ Outline VPN ที่มีอยู่แทนที่จะใช้การปรับใช้แบบกำหนดเองที่ซับซ้อน คุณก็ทำได้ โดยสามารถดึงข้อมูลการเข้าถึงจากคีย์การเข้าถึงที่มีอยู่ ตัวอย่างด้านล่างนี้แสดงให้เห็นว่าคุณสามารถค้นหาข้อมูลที่เกี่ยวข้องแต่ละส่วนได้จากที่ใดบ้าง ได้แก่ "server", "server_port", "password" และ "method"

ตัวอย่างคีย์การเข้าถึงแบบคงที่

ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

  1. เซิร์ฟเวอร์

    ตำแหน่ง "server" คือส่วนของคีย์การเข้าถึงแบบคงที่ระหว่างสัญลักษณ์ "@" และ ":" ต่อไปนี้

    ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

  2. พอร์ตของเซิร์ฟเวอร์

    ตำแหน่ง "server_port" คือส่วนของคีย์การเข้าถึงแบบคงที่ระหว่างสัญลักษณ์ ":" ที่สองและ "/" ต่อไปนี้

    ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

  3. รหัสผ่าน

    ในส่วนนี้อาจมีความซับซ้อนเล็กน้อย ส่วนของคีย์ที่อยู่ระหว่าง "ss://" และ "@" จะเข้ารหัสในรูปแบบ base64 ซึ่งคุณสามารถใช้เครื่องมือการถอดรหัส base64 เพื่อแสดงค่าที่ซ่อนอยู่ในส่วนนี้

    ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

    ในกรณีนี้ เมื่อเรายกเลิกการเข้ารหัสส่วนนี้ของคีย์ เราจะได้ผลลัพธ์ดังด้านล่าง โดยส่วนที่เป็นตัวหนา (หลังเครื่องหมาย ":") คือ "password"

    chacha20-ietf-poly1305:5REfxTjltz2L

  4. วิธีการ

    และสุดท้ายส่วนที่อยู่ด้านหน้า ":" ของผลลัพธ์เดียวกันกับข้างต้นก็คือ "method"

    chacha20-ietf-poly1305:5REfxTjltz2L

ตัวอย่างคีย์การเข้าถึงแบบไดนามิก

เมื่อพิจารณาทั้งหมดแล้ว ข้อมูลการเข้าถึงขั้นสุดท้ายแบบ JSON สำหรับคีย์ตัวอย่างจะมีลักษณะดังนี้

การสร้างคีย์การเข้าถึงแบบไดนามิก

เมื่อมี URL ที่จัดเก็บข้อมูลการเข้าถึงแล้ว สิ่งที่คุณต้องทำเพื่อสร้างคีย์การเข้าถึงแบบไดนามิกก็คือการแทนที่โปรโตคอล https:// ด้วยโปรโตคอล ssconf:// ซึ่งจะเป็นการบอกวิธีจัดการคีย์การเข้าถึงนี้ให้กับไคลเอ็นต์ Outline

ก่อน (โปรโตคอล https) https://s3.amazonaws.com/a-cool-s3-bucket/my-access-key.json

หลัง (คีย์การเข้าถึงแบบไดนามิกที่มีโปรโตคอล ssconf) ssconf://s3.amazonaws.com/a-cool-s3-bucket/my-access-key.json

คุณสามารถเพิ่มคีย์นี้ไปยังไคลเอ็นต์ Outline ได้เลย
