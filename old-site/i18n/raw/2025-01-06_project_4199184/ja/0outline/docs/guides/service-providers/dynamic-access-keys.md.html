バージョン 1.8.0 以降では、Outline クライアントで動的アクセスキーのベータ版がサポートされています。以前は、Outline VPN にアクセスするために必要な情報がアクセスキー自体に（静的に）保存されていました。しかし、動的アクセスキーを使用すると、その情報をリモート ロケーションでホストし、動的に更新することができます。

つまり、すべてのクライアント ユーザーにアクセスキーを再発行しなくても、VPN のロケーション、それがホストされるポート、アクセスするためのパスワード、使用される暗号化方式をすべてその場で切り替えることができます。

アクセス情報を ss:// リンクとしてホストする

Outline クライアント バージョン 1.8.1 以降では、動的アクセスキー リンクで保存されるアクセス情報を単純に ss:// リンクにすることができます（以前は、下記のように、動的アクセスキーは JSON オブジェクトのみでした）。サーバー、ポート、暗号化方式は動的に変更する予定がなくても、接続先のサーバーはその場の状況に応じて変更したいユーザーには、この方法が適しています。ss キーを JSON オブジェクトに変換する必要がなく、動的キーを Google スプレッドシートに保存できるという利点があります。これは、Google スプレッドシートが利用可能な場合に、検閲耐性のあるオプションとなります。

  • アクセス情報を Google スプレッドシートに保存する
    • Google アカウントを作成します。
    • https://sheets.new で新しいスプレッドシートを開きます。
    • ss:// リンクをセル A1 に貼り付けます。
    • [ファイル] > [共有] > [ウェブに公開] を選択します。
    • [リンク] タブで [シート1] と [カンマ区切り形式（.csv）] を選択します。
    • [公開] をクリックします。
    • リンクは次のようになります: https://docs.google.com/spreadsheets/d/e/2PACX-1vTN0nxRcUf52R6fq3JdVfNqIPr2GQU2wBGsG-xI4WTaYrliCElxp0ZAAjy0PDlf_-qvCT3pnSB2jVEo/pub?gid=0&single=true&output=csv
    • https:// を ssconf:// に置換して動的アクセスキーを作成します。キーは次のようになります: ssconf://docs.google.com/spreadsheets/d/e/2PACX-1vTN0nxRcUf52R6fq3JdVfNqIPr2GQU2wBGsG-xI4WTaYrliCElxp0ZAAjy0PDlf_-qvCT3pnSB2jVEo/pub?gid=0&single=true&output=csv
  • アクセス情報を保存するための、検閲耐性のあるその他のオプションは、下記のとおりです。

アクセス情報を JSON オブジェクトとしてホストする

アクセス情報は、以下のキーを含む JSON オブジェクトとして保存することもできます。

  • server: VPN がホストされているサーバーのドメインまたは IP アドレス。二重引用符で囲みます。

  • server_port: VPN サービスが実行されているポート。

    デフォルトでは、Outline マネージャーは、サーバーで実行されている他のサービスなどへの干渉を避けるために、エフェメラル ポートで VPN をホストします。可能であれば、ポートスキャナによって簡単に検出されないように、標準ポート（443 など）で VPN をホストすることを検討してください。

  • password: VPN にアクセスするために必要なシークレット フレーズまたはトークン。二重引用符で囲みます。たとえば、"Open Sesame!" などです。

    パスワードについては、エントロピーが高いほど強力です。20 文字以上のパスワードを総当たり攻撃で特定するには、平均で、宇宙の一生ほどの年月がかかります。

  • method: VPN で使用する暗号化方式。二重引用符で囲みます。サポートされている方式の詳細は、Shadowsocks のウェブサイトで確認できます。

現時点では、ブロックされないアクセス情報を保存する場所は VPN マネージャーが選択します。以下に、選択可能な、検閲耐性のあるオプションの例をいくつか示します。

  • pad.riseup.net

    • このオプションを使用する場合、新しいアカウントを作成する必要はありません。
    • ファイルを作成した後、右上の [Share] ボタンをクリックし、[Read only] をオンにします。リンクをコピーします。リンクは次のようになります: https://pad.riseup.net/p/r.9b3d9974b0fd86132c872eb8e5c84c24
    • リンクの末尾に /export/txt を追加します。リンクは次のようになります: https://pad.riseup.net/p/r.9b3d9974b0fd86132c872eb8e5c84c24/export/txt
    • https:// を ssconf:// に置換して動的アクセスキーを作成します。キーは次のようになります: ssconf://pad.riseup.net/p/r.9b3d9974b0fd86132c872eb8e5c84c24/export/txt

  • S3（必ずパススタイルのアクセスを使用してください）

    • AWS アカウントを作成し、新しいバケットを作成します。
    • バケットの [Permissions] 設定のオブジェクト所有権で、[ACLs enabled] と [Bucket owner preferred] を選択します。
    • アクセス コントロール リスト（ACL）で、「Everyone (public access)」のバケット ACL に「Read」アクセスを割り当てます。
    • JSON ファイルをバケットに追加します（ヒント: JSON ファイルはテキスト エディタで作成し、拡張子を .json として保存します）。
    • ファイルが表示可能かを確認するために、オブジェクトを選択し、[Copy URL] を選択して、シークレット モードのブラウザ ウィンドウでテストします。URL は次のようになります: https://my-bucket-name.s3.us-east-2.amazonaws.com/my_online_config_file.json
    • https:// を ssconf:// に置換して動的アクセスキーを作成します。キーは次のようになります: ssconf://my-bucket-name.s3.us-east-2.amazonaws.com/my_online_config_file.json

  • Netlify

独自の動的キー ソリューションを展開する

動的キーを自分でホストする場合は、必ずクロスオリジン リクエスト（「CORS」とも呼ばれます）を有効にする必要があります。これを行うには、ワイルドカード値（"*" など）を使用して、Access-Control-Allow-Origin ヘッダーをすべてのサーバー レスポンスに追加し、すべてのユーザーが動的キーにアクセスできるようクライアントに命令します。

静的アクセスキーからアクセス情報を取得する

まだ使い始めたばかりで、手の込んだカスタム デプロイメントではなく、既存の Outline VPN に間接的なレイヤを追加したい場合でも、心配は不要です。既存のアクセスキーからアクセス情報を抽出できます。次の例は、"server"、"server_port"、"password"、"method" という各関連情報がどの部分にあるかを示しています。

静的アクセスキーの例:

ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

  1. サーバー

    "server" は、静的アクセスキーのうち "@" 記号とその後の ":" の間にある部分です。

    ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

  2. サーバー ポート

    "server_port" は、静的アクセスキーのうち 2 番目の ":" 記号とその後の "/" の間にある部分です。

    ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

  3. パスワード

    "password" は若干わかりにくいです。キーのうち "ss://" と "@" の間にある部分ですが、この部分は base64 形式でエンコードされています。base64 デコーダを使用すると、この部分の内容を確認できます。

    ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

    この例では、キーのこの部分をデコードすると、次に示す結果が得られます。太字の部分（":" より後の部分）が "password" です。

    chacha20-ietf-poly1305:5REfxTjltz2L

  4. 方式

    ご推察のとおり、上記の結果のうち、":" より前の部分が "method" です。

    chacha20-ietf-poly1305:5REfxTjltz2L

動的アクセスキーを作成する

アクセス情報の保存場所への URL を取得したら、あとは https:// プロトコルを ssconf:// プロトコルに置換すれば動的アクセスキーを作成できます。これは、このアクセスキーをどのように処理すべきかを Outline クライアントに指示します。

置換前（https プロトコル） https://s3.amazonaws.com/a-cool-s3-bucket/my-access-key.json

置換後（ssconf プロトコルを使用した動的アクセスキー） ssconf://s3.amazonaws.com/a-cool-s3-bucket/my-access-key.json

このキーを Outline クライアントに追加できます。

カスタムエラー

クライアント バージョン 1.13 以降では、動的アクセスキーを介してカスタムのエラー メッセージをエンドユーザーに送信できるようになりました。これを行うには、エラーキーを JSON に追加します。

次回エンドユーザーが接続するときに、接続が失敗すると、画面の下部にこのエラー メッセージが表示されます。
