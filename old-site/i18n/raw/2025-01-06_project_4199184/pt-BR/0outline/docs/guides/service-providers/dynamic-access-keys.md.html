A partir da versão 1.8.0 (Beta), o app cliente do Outline pode usar chaves de acesso dinâmico. Antes, as informações necessárias para acessar uma VPN do Outline VPN ficavam armazenadas (estaticamente) na própria chave de acesso. Entretanto, com as chaves de acesso dinâmicas, é possível hospedar essas informações em um local remoto e atualizá-las durante o uso.

Isso significa que você pode mudar em tempo real o local da VPN, a porta em que ela está hospedada, a senha para acessá-la e o método de criptografia que está sendo usado. Tudo isso sem precisar gerar novas chaves de acesso para todos os usuários clientes.

Como hospedar informações de acesso como um link ss://

A partir da versão 1.8.1, as informações de acesso armazenadas no link da chave de acesso dinâmica podem ser um simples link ss://. Antes, a chave dinâmica só podia ser um objeto JSON, conforme descrito abaixo. Essa solução é adequada para pessoas que não planejam alterar dinamicamente o servidor, a porta ou o método de criptografia, mas querem poder mudar em tempo real o servidor a que estão conectadas. Dessa forma, não é preciso converter a chave ss em um objeto JSON. Outra vantagem é que a chave dinâmica pode ser armazenada em planilhas Google, uma opção para contornar a censura em locais onde é possível acessar o app Planilhas Google.

  • Com armazenar informações de acesso em planilhas Google
    • Crie uma conta do Google.
    • Abra uma planilha em https://sheets.new.
    • Cole o link ss:// na célula A1.
    • Selecione Arquivo > Compartilhar > Publicar na Web.
    • Na guia "Link", selecione as opções "Planilha1" e "Valores separados por vírgulas (.csv)".
    • Clique em "Publicar".
    • O link terá esta aparência: https://docs.google.com/spreadsheets/d/e/2PACX-1vTN0nxRcUf52R6fq3JdVfNqIPr2GQU2wBGsG-xI4WTaYrliCElxp0ZAAjy0PDlf_-qvCT3pnSB2jVEo/pub?gid=0&single=true&output=csv.
    • Substitua https:// por ssconf:// para criar a chave de acesso dinâmica, que deve ser algo como: ssconf://docs.google.com/spreadsheets/d/e/2PACX-1vTN0nxRcUf52R6fq3JdVfNqIPr2GQU2wBGsG-xI4WTaYrliCElxp0ZAAjy0PDlf_-qvCT3pnSB2jVEo/pub?gid=0&single=true&output=csv

Como hospedar informações de acesso como um objeto JSON

Outra alternativa é armazenar as informações de acesso como um objeto JSON contendo as seguintes chaves:

  • server: o domínio ou endereço IP do servidor em que sua VPN está hospedada, entre aspas duplas.

  • server_port: a porta em que seu serviço VPN está funcionando.

    Por padrão, o Outline Manager hospeda a VPN em uma porta efêmera para evitar interferência em outras operações que estejam em execução no servidor. Se possível, hospede a VPN em uma porta padrão (como 443) para evitar que ela seja facilmente detectada por um scanner de porta.

  • password: a frase secreta ou o token necessário para entrar na VPN, entre aspas duplas. Algo como "Abre-te Sesamo!"

    No caso das senhas, quanto mais complexidade, maior a segurança. Uma senha com 20 caracteres ou mais levaria, em média, o tempo de existência do universo para ser descoberta.

  • method: o método de criptografia usado pela VPN, entre aspas duplas. Saiba mais sobre os métodos que podem ser usados no site do Shadowsocks.

Como lançar a solução de chave dinâmica

Se você optar por hospedar a chave dinâmica por conta própria, verifique se ativou a solicitação entre origens (também conhecida como "CORS"). Para fazer isso, adicione o cabeçalho Access-Control-Allow-Origin a todas as respostas do servidor com um caractere curinga ("*"). Com isso, o cliente entenderá que qualquer pessoa pode acessar sua chave dinâmica.

Como receber informações de acesso da chave de acesso estática

Se estiver apenas começando e quiser adicionar uma camada extra de indireção à VPN do Outline, em vez de fazer uma implantação personalizada sofisticada, não se preocupe. Você pode extrair as informações de acesso de uma chave de acesso já criada. O exemplo abaixo mostra onde encontrar cada uma das informações: "server", "server_port", "password" e "method".

Exemplo de chave de acesso estática:

ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

  1. Servidor

    O local "server" é a parte da chave de acesso estática entre o símbolo "@" e ":" a seguir.

    ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

  2. Porta do servidor

    O local "server_port" é a parte da chave de acesso estática entre o segundo caractere ":" e "/" a seguir.

    ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

  3. Senha

    É aqui que as coisas ficam um pouco mais complexas. A parte da chave que fica entre "ss://" e "@" é codificada no formato base64. Use um decodificador de base64 para saber o que está oculto nesta parte:

    ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

    No nosso caso, quando decodificamos essa parte da chave, chegamos ao resultado abaixo, em que a parte em negrito (depois de ":") é "password":

    chacha20-ietf-poly1305:5REfxTjltz2L

  4. Método

    E, como talvez você tenha descoberto, a parte do mesmo resultado acima antes de ":" é "method".

    chacha20-ietf-poly1305:5REfxTjltz2L

Como criar a chave de acesso dinâmica

Depois que você tiver o URL do local onde as informações de acesso estão armazenadas, só é preciso substituir o protocolo https:// pelo ssconf:// para criar a chave de acesso dinâmica. Isso informa o app cliente do Outline como lidar com essa chave de acesso específica.

ANTES (protocolo https) https://s3.amazonaws.com/a-cool-s3-bucket/my-access-key.json

DEPOIS (chave de acesso dinâmica com o protocolo ssconf) ssconf://s3.amazonaws.com/a-cool-s3-bucket/my-access-key.json

Tudo pronto para você adicionar essa chave ao app cliente do Outline!
