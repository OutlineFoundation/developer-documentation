Outline을 사용하기 전에 핵심 개념을 살펴보는 것이 좋습니다.

Outline의 작동 원리

  1. Outline Manager: 클라우드 제공업체에서 또는 자체 하드웨어에서 Outline 서버를 만들고 액세스 키를 생성합니다.
  2. 키 배포: 이러한 키를 친구나 가족과 공유합니다.
  3. 터널 트래픽: 사용자가 Outline 클라이언트를 설치하고 키를 사용하여 안전한 터널을 설정해 검열을 우회합니다.

액세스 키

액세스 키는 사용자 인증 정보로, 이를 통해 사용자가 Outline VPN 서버에 연결할 수 있습니다. 액세스 키에는 Outline 클라이언트가 보안 연결을 설정하는 데 필요한 정보가 포함됩니다. 액세스 키에는 정적 액세스 키와 동적 액세스 키라는 두 가지 유형이 있습니다.

정적 액세스 키

정적 액세스 키에서 서버 액세스 정보는 액세스 키에 직접 인코딩됩니다. 이렇게 하면 액세스 정보를 수정할 수 없습니다. 사용자는 이 키를 Outline 클라이언트로 가져와 서버에 연결합니다.

  • 예: ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

동적 액세스 키

동적 액세스 키는 서버 액세스 정보의 위치를 인코딩하므로 사용자 액세스를 특정 서버 세부정보에서 분리할 수 있습니다. 즉, 이미 배포한 액세스 키를 업데이트하지 않고도 서버의 IP 주소와 포트, 보안 설정 등을 변경할 수 있습니다.

자세한 내용은 동적 액세스 키에 관한 문서를 참고하세요.
