Outline offre deux types de clés d'accès : statique et dynamique. Les clés statiques encodent directement toutes les informations de connexion, tandis que les clés dynamiques encodent l'emplacement des informations de connexion, ce qui vous permet de stocker ces informations à distance et de les modifier si nécessaire. Vous pouvez ainsi modifier la configuration de votre serveur sans avoir à générer et distribuer de nouvelles clés à vos utilisateurs. Ce document explique comment utiliser des clés d'accès dynamiques pour une gestion plus flexible et efficace de votre serveur Outline.

Pour indiquer les informations d'accès qui seront utilisées par vos clés d'accès dynamiques, vous pouvez suivre l'une de ces trois méthodes :

Utiliser un lien ss://

Cette méthode vous permet de gérer tous les aspects de la connexion Outline de vos utilisateurs avec plus de flexibilité. Vous pouvez ainsi modifier le serveur, le port, le mot de passe et la méthode de chiffrement.

Exemple :

  • server : domaine ou adresse IP de votre serveur VPN
  • server_port : numéro du port sur lequel le serveur VPN est exécuté
  • password : mot de passe requis pour se connecter au VPN
  • method : méthode de chiffrement utilisée par le VPN – voici les algorithmes de chiffrement AEAD acceptés par Shadowsocks

Utiliser un objet YAML

Client Outline v1.15.0 et versions ultérieures

Cette méthode est semblable à la méthode JSON décrite précédemment, mais donne encore plus de flexibilité grâce au format de configuration avancée d'Outline. Vous pouvez modifier le serveur, le port, le mot de passe, la méthode de chiffrement et bien plus encore.

Exemple :

  • transport : définit les protocoles de transport à utiliser (TCP et UDP dans ce cas)
  • tcp/udp : précise la configuration pour chaque protocole
    • $type : indique le type de configuration, shadowsocks ici
    • endpoint : domaine ou adresse IP et port de votre serveur VPN
    • secret : mot de passe requis pour se connecter au VPN
    • cipher : méthode de chiffrement utilisée par le VPN – voici les algorithmes de chiffrement AEAD acceptés par Shadowsocks

Extraire des informations d'accès d'une clé statique

Si vous avez déjà une clé d'accès statique, vous pouvez en extraire les informations pour créer une clé d'accès dynamique basée sur JSON ou YAML. Les clés d'accès statiques sont configurées comme ceci :

Exemple :

ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTpleGFtcGxl@outline-server.example.com:8388/?outline=1

  • Serveur : outline-server.example.com
  • Port du serveur : 8388

  • Informations utilisateur : Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTpleGFtcGxl décodé en base64 à l'aide d'un outil comme Google Admin Toolbox Encode/Decode

    • Méthode : chacha20-ietf-poly1305
    • Mot de passe : example

Choisir une plate-forme d'hébergement

Maintenant que vous savez comment créer des clés d'accès dynamiques, il est important de choisir une plate-forme d'hébergement adaptée à la configuration de votre clé d'accès. Pour sélectionner la bonne plate-forme, il faut prendre en compte sa fiabilité, le degré de sécurité qu'elle fournit, sa facilité d'utilisation et sa résistance à la censure. Fournira-t-elle systématiquement vos informations de clé d'accès sans temps d'arrêt ? Offre-t-elle les mesures de sécurité adaptées pour protéger votre configuration ? Permet-elle de gérer vos informations de clé d'accès facilement ? Est-elle accessible dans les régions où des politiques de censure sur Internet sont en vigueur ?

Pour les situations où l'accès aux informations peut être restreint, vous pouvez héberger votre serveur sur des plates-formes résistantes à la censure comme Google Drive, pad.riseup.net, Amazon S3 (avec un accès de type chemin d'accès), Netlify ou les gists secrets GitHub. Évaluez les besoins spécifiques de votre déploiement et choisissez une plate-forme adaptée à vos impératifs en termes d'accessibilité et de sécurité.
