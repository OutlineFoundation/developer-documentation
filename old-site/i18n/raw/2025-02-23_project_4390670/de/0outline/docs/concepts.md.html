Es könnte hilfreich sein, wenn Sie sich vor der Verwendung von Outline mit den Grundlagen befassen.

So funktioniert Outline

  1. Outline-Manager: Sie erstellen einen Outline-Server – entweder auf einem Cloud-Anbieter oder auf Ihrer eigenen Hardware – und generieren dann Zugriffsschlüssel.
  2. Schlüssel bereitstellen: Sie geben diese Schlüssel an Freunde und Familie weiter.
  3. Tunnel-Traffic: Nutzer installieren den Outline-Client und nutzen Schlüssel, um einen sicheren Tunnel einzurichten und Zensur zu umgehen.

Zugriffsschlüssel

Die Zugriffsschlüssel sind die Anmeldedaten, mit denen Nutzer eine Verbindung zum VPN auf Ihrem Outline-Server herstellen können. Sie enthalten alle Informationen, die der Online-Client benötigt, um eine verschlüsselte Verbindung aufzubauen. Es gibt zwei Arten von Zugriffsschlüsseln: statische und dynamische.

Statische Zugriffsschlüssel

Beim statischen Zugriffsschlüssel befinden sich die Zugriffsinformationen direkt im Schlüssel. So können sie nicht geändert werden. Nutzer importieren diesen Schlüssel in den Outline-Client, um sich mit dem Server zu verbinden.

  • Beispiel: ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

Dynamische Zugriffsschlüssel

Dynamische Zugriffsschlüssel enthalten lediglich den Speicherort der Informationen für den Serverzugriff. So haben Sie die Möglichkeit, bestimmte Serverdetails vom Nutzerzugriff abzukoppeln. Das heißt, Sie können die IP-Adresse, den Port, die Sicherheitseinstellungen und andere Einstellungen Ihres Servers ändern, ohne die bereits verteilten Zugriffsschlüssel aktualisieren zu müssen.

Weitere Informationen finden Sie in der ausführlichen Dokumentation zu dynamischen Zugriffsschlüsseln.
