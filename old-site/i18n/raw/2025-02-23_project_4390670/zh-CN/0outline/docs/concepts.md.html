使用 Outline 前，首先来了解一些核心概念会非常有帮助。

Outline 的工作原理

  1. Outline 管理器：您通过云服务提供商或自己的硬件创建 Outline 服务器，然后生成访问密钥。
  2. 分发密钥：您将这些密钥共享给亲朋好友。
  3. 隧道流量：用户安装 Outline 客户端，并使用密钥建立安全隧道，绕开审查。

访问密钥

访问密钥是允许用户连接到您的 Outline VPN 服务器的凭据。它们包含 Outline 客户端建立安全连接所需的信息。访问密钥分为两种类型：静态和动态。

静态访问密钥

在静态访问密钥中，服务器访问信息直接编码在访问密钥中。这可以防止访问信息被修改。用户将此密钥导入 Outline 客户端以连接到服务器。

  • 示例： ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTo1UkVmeFRqbHR6Mkw@outline-server.example.com:17178/?outline=1

动态访问密钥

动态访问密钥对服务器访问信息的位置进行编码，让您能够将用户访问与特定服务器详细信息分离。这意味着您可以更改服务器的 IP 地址、端口、安全设置等，而无需更新已分发的访问密钥。

如需了解详情，请参阅关于动态访问密钥的详细文档。
