Outline 提供两种类型的访问密钥：静态和动态。静态密钥对密钥本身内的所有连接信息进行编码，而动态密钥对连接信息的位置进行编码，这可让您远程存储这些信息，并根据需要进行更改。这意味着您可以更新服务器配置，而无需生成新密钥并将其分发给用户。本文档介绍了如何使用动态访问密钥来更灵活、更高效地管理 Outline 服务器。

您可以通过三种格式来指定动态访问密钥将使用的访问信息：

使用 ss:// 链接

使用 JSON 对象

Outline 客户端 1.8.0 及以上版本。

这种方式可以让您更灵活地管理用户 Outline 连接的各个方面。您可以通过这种方式更新服务器、端口、密码和加密方法。

示例：

  • server：VPN 服务器的域名或 IP 地址。
  • server_port：运行 VPN 服务器的端口号。
  • password：连接到 VPN 所需的密码。
  • method：VPN 所用的加密方法。请参阅 Shadowsocks 支持的 AEAD 加密。

使用 YAML 对象

Outline 客户端 1.15.0 及以上版本。

此方式与前面的 JSON 方式类似，但通过利用 Outline 的高级配置格式增加了更多灵活性。您可以更新服务器、端口、密码、加密方法等等。

示例：

  • transport：定义要使用的传输协议（本例中为 TCP 和 UDP）。
  • tcp/udp：指定每种协议的配置。
    • $type：表示配置的类型，本例中为 Shadowsocks。
    • endpoint：VPN 服务器的域名或 IP 地址和端口。
    • secret：连接到 VPN 所需的密码。
    • cipher：VPN 所用的加密方法。请参阅 Shadowsocks 支持的 AEAD 加密。

如需详细了解配置 Outline 服务器访问权限的所有方式（包括传输、端点、拨号器和数据包监听器），请参阅访问密钥配置。

  • 服务器端口：8388

    • 方法：chacha20-ietf-poly1305

选择托管平台

现在您已经了解了如何创建动态访问密钥，为访问密钥配置选择合适的托管平台非常重要。在选择托管平台时，务必要考虑平台的可靠性、安全性、易用性和抗审查性等因素。平台能否始终如一地为您提供访问密钥信息而不会停机？平台是否会提供适当的安全措施来保护您的配置？在平台上管理访问密钥信息的难易程度如何？平台能否在实行网络审查制度的地区访问？

如果访问信息可能会受到限制，不妨考虑托管在具有抗审查性的平台上，例如 Google 云端硬盘、pad.riseup.net、Amazon S3（提供路径式访问）、Netlify 或 GitHub 机密 Gist。评估部署的具体需求，并根据您对可访问性和安全性的要求选择合适的平台。
